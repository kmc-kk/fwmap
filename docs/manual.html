fwmap マニュアル

fwmap は、組込みファームウェアの ELF と GNU ld 系 map を解析し、ROM/RAM 使用量、主要シンボル、object 寄与、前回ビルドとの差分を可視化するローカル CLI ツールです。

現行版は Phase 3 まで実装済みで、単純なサイズ表示だけでなく、差分原因の追跡を主機能として扱います。

1. 概要

  • どの section が ROM / RAM を使っているか
  • どの symbol が大きいか
  • どの object file がサイズに効いているか
  • 前回ビルドから何が増えたか、減ったか
  • 追加・削除・増加・減少のどれに当たるか

2. 対応範囲

  • ELF32 / ELF64
  • little-endian ELF を主対象
  • GNU ld 系 map
  • 単一 HTML レポート出力
  • 前回成果物との diff 比較
  • 固定しきい値ベースの warning
  • --verbose / --version

未対応または限定的な内容: linker script 解析、memory region 可視化、JSON 出力、CI 向け exit code 制御、demangle 改善、DWARF を用いたソース行解析。

3. ビルドとテスト

前提:

  • Rust toolchain
  • Windows / Linux / macOS
cargo build
cargo test

4. 基本的な使い方

ELF のみ解析

fwmap analyze --elf build/app.elf

ELF と map を解析

fwmap analyze --elf build/app.elf --map build/app.map --out report.html

前回ビルドとの差分を解析

fwmap analyze --elf build/app.elf --map build/app.map --prev-elf prev/app.elf --prev-map prev/app.map --out report.html

補助オプション

fwmap --version
fwmap analyze --elf build/app.elf --verbose

--out を省略した場合は fwmap_report.html が出力されます。

5. CLI オプション

オプション必須説明
--elf <path>必須現在の ELF ファイル
--map <path>任意現在の GNU ld map ファイル
--prev-elf <path>任意比較用の前回 ELF
--prev-map <path>任意比較用の前回 map
--out <path>任意HTML 出力先
--verbose任意warning 詳細を標準出力へ表示
--version任意バージョン表示
--help任意ヘルプ表示

6. 出力内容

fwmap は標準出力の実行サマリと、単一 HTML レポートを出力します。

ELF: build/app.elf
ROM: 32768 bytes (32.00 KiB) | RAM: 8192 bytes (8.00 KiB) | Sections: 24 | Symbols: 180 | Warnings: 2
ROM: +1024 bytes
RAM: +256 bytes
Diff counts: sections +1 / -0 / ↑3 / ↓1, symbols +2 / -1 / ↑4 / ↓3
Top growth symbol: foo_bar (+512)
Top growth object: drivers/net.o (+768)
Report: fwmap_report.html

7. HTML レポートの見方

  1. Header
  2. Overview
  3. Warnings
  4. Memory Summary
  5. Section Breakdown
  6. Top Symbols
  7. Top Object Contributions
  8. Diff
  9. Footer

Overview

対象 ELF パス、アーキテクチャ、ELF class、endian、section 数、ROM 合計、RAM 合計、warning 件数、diff がある場合は ROM/RAM 差分を表示します。

Warnings

warning は source と関連対象付きで表示されます。ROM/RAM 使用率超過、巨大 symbol、.data / .bss 増加、symbol table 欠損、map の一部読み飛ばしなどが含まれます。

Memory Summary

section をサイズ順に表示し、ROM / RAM / Other に分類します。

Section Breakdown

section ごとの名前、アドレス、サイズ、flags を表示します。

Top Symbols

ELF symbol table からサイズ上位を表示します。symbol table が無い場合でも HTML 生成自体は継続します。

Top Object Contributions

map がある場合に object ごとの寄与サイズ上位を表示します。

Diff

Phase 3 で最も強化されたセクションです。ROM/RAM 差分、section/symbol/object の変化件数、上位増加一覧、Added / Removed 一覧を表示します。

差分分類は Added, Removed, Increased, Decreased, Unchanged, Moved です。現行版では主に Added / Removed / Increased / Decreased / Unchanged を使います。

8. graceful degradation

  • --map が無くても HTML は生成可能
  • symbol table が無くても section summary は生成可能
  • map に壊れた行があっても warning 化して継続

一方で、ファイル不存在、非 ELF、section table 欠損、HTML 出力先への書き込み失敗などは停止要因です。

9. エラーメッセージ

主なエラーは、ELF / map ファイル不存在、非 ELF 入力、未対応 ELF metadata、section table 欠損、HTML レポート書き込み失敗です。CLI は panic を直接見せない方針です。

10. 典型的な使い方

まず ELF だけ見る

fwmap analyze --elf build/app.elf

object 寄与を見る

fwmap analyze --elf build/app.elf --map build/app.map

回帰調査をする

fwmap analyze --elf current/app.elf --map current/app.map --prev-elf previous/app.elf --prev-map previous/app.map --out diff_report.html

11. 内部構成

  • cli: 引数処理と実行制御
  • ingest: ELF / map の読み込み
  • analyze: 集計と warning 判定
  • diff: 差分計算と分類
  • model: 共通データ構造
  • render: CLI / HTML 出力

12. 既知の制約

  • ELF は現在 SHT_SYMTAB を中心に参照
  • map は GNU ld の典型出力を優先
  • object path は主に map 由来
  • archive/member の表記揺れは主要ケース対応に留まる
  • linker script 非対応のため region 単位の厳密解析は未実装
  • ROM/RAM はヒューリスティック集計
